One panel, five providers, zero config drift

Every API call routes through a single service worker that translates your prompt into each provider's native format. Responses are normalized so the UI never needs to know which model answered.

background.js 
// Normalized response shape — every provider
function normalize(provider, raw) {
  const text = extractors[provider](raw);
  return {
    content: [{
      type: 'text',
      text
    }]
  };
}

// Route to active provider
const handlers = {
  anthropic: callAnthropic,
  openai:    callOpenAI,
  gemini:    callGemini,
  openrouter:callOpenRouter,
  ollama:    callOllama
};

Up and running in 4 steps

1

Install the extension

Unzip the release into a folder. Open vivaldi://extensions, enable Developer Mode, and load unpacked.

2

Choose your provider

Open Options, pick a provider chip, enter your API key, and select a model from the dropdown (or type a slug for OpenRouter/Ollama).

3

Test the connection

Hit "Test Connection" to fire a real API ping through the service worker. Green means you're forging.

4

Start forging

Open the panel, type a prompt, and hit Enhance. A/B test it. Save it to your library. Switch providers and try again. Forge until it's perfect.
