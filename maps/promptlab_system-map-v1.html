User Interface Layer
panel.html
Entry point. Loads compiled React bundle from assets/panel-*.js. Served via chrome-extension:// or as Vivaldi Web Panel.
options.html
Vanilla HTML/JS settings page. Independent of Vite build. Provider chips, API key fields, model selectors, Test Connection button. Always up to date — no build step needed.
↕ chrome.runtime.sendMessage({ type: 'ANTHROPIC_REQUEST' })
React Application (compiled into assets/panel-*.js)
App.jsx ~2000 lines
Monolith component. Contains: Editor, Composer, A/B Test, Library, Pad, Wizard, Settings modal, command palette. 40+ useState hooks. Biggest maintenance risk.
api.js
Thin bridge. Extension mode → sendMessage. Web mode → direct Anthropic fetch (Anthropic only). Exports callAnthropic() — name is misleading, routes to any provider via background.
errorTaxonomy.js
Normalizes errors into user-friendly messages with suggestions and action buttons. Built for Anthropic error shapes; other providers get generic fallback.
promptUtils.js
Pure functions: wordDiff, scorePrompt, extractVars, encodeShare, extractTextFromAnthropic, parseEnhancedPayload, normalizeLibrary.
promptLint.js
Static analysis of prompt text. Checks for missing role, goal, constraints, output format, examples. Quick-fix templates.
sensitiveData.js
Detects API keys, emails, credit cards in prompt text. Pattern matching + configurable settings. Loaded from pl2-redaction-settings.
redactionGate.js
Applies redaction: replaces detected sensitive data with placeholders before sending to provider. Works on any payload shape.
experimentHistory.js
CRUD for A/B test experiment records. Max 500. Stores in pl2-experiment-history. Includes hash, timestamp, winner, notes.
experimentStore.js
IndexedDB-backed persistence with localStorage fallback. Used by experiment history for durability. DB: prompt_lab_local.
icons.jsx
Inline SVG icon component. Replaces lucide-react to avoid dependency. ~30 icons as path data.
↕ chrome.runtime.sendMessage / sendResponse
Background Service Worker (Credential Isolation)
background.js
The heart of multi-provider routing. Reads provider from chrome.storage.local, routes to the correct call*() function.

5 providers: callAnthropic, callOpenAI, callGemini, callOpenRouter, callOllama

Key job: Translate Anthropic-shaped payload → provider-native format → normalize response to { content: [{ type: 'text', text }] }
options.js
Reads/writes provider config to chrome.storage.local. Fires test connectivity pings through the service worker. Vanilla JS — no build step.
↕ chrome.storage.local (credentials) · ↕ fetch() (API calls)
Persistence
chrome.storage.local
Provider, API keys, models, Ollama URL. Only accessible from background.js and options.js. Never exposed to panel context.
localStorage (panel)
Library, collections, experiments, redaction settings, pad text, onboarding flag, dark mode. Accessible from React panel only.
IndexedDB
Experiment store fallback. DB: prompt_lab_local, store: experiments. Used when localStorage is unreliable.
External APIs
api.anthropic.com
Anthropic Messages API. Header auth (x-api-key). Payload passthrough — native format.
api.openai.com
OpenAI Chat Completions. Header auth (Bearer). Needs message format conversion.
generativelanguage.googleapis.com
Gemini v1beta. Key-in-URL auth (query param). Needs contents/parts conversion + systemInstruction.
openrouter.ai
OpenAI-compatible gateway. Header auth. Same converter as OpenAI. Free-text model slug.
localhost:11434
Ollama local. No auth. Same message format as OpenAI. Needs stream: false.
Build Pipeline
vite.config.js
Vite + React plugin. Outputs to dist/. Base set to ./ for chrome-extension:// paths.
assemble.js
Post-build script. Copies extension/ files (manifest, background, options, icons) into dist/ alongside Vite output.
tailwind + postcss
Tailwind CSS compiled into panel stylesheet. Used by App.jsx utility classes.
gen_icons.py
Generates extension icon PNGs (16, 48, 128) from a template. Run once during setup.

Issues & Improvement Opportunities

Ordered by impact. Red = bugs or drift risks. Amber = maintenance drag. Green = nice-to-have cleanup.

High Compiled panel assets are stale
The assets/panel-*.js files in the built extension are from the pre-multi-provider Vite build. The source App.jsx has wizardModel in enhance() and the 5-provider dropdown, but none of that is in the compiled JS the user actually loads. The Options page and background.js are ahead of the panel.
Fix → Run npm run build in prompt-lab-source and copy dist output to prompt-lab-extension/assets/
High anthropicModel saved but never read
The Options page writes anthropicModel to chrome.storage.local, but background.js never reads it — it passes the payload's model field directly to the Anthropic API. Every other provider reads its model from storage. This means changing the Anthropic model in Options has no effect.
Fix → In background.js callAnthropic, read store.anthropicModel and use it to override payload.model if present
High Duplicate message converters will silently diverge
toOllamaMessages() and toOpenAIMessages() in background.js are identical functions. If one is updated (e.g., to handle tool_use blocks) and the other isn't, Ollama and OpenAI will behave differently with no warning.
Fix → Merge into a single toChatMessages() function used by both callOllama and callOpenAI
Medium Nested duplicate directories in source
prompt-lab-source/prompt-lab-extension/ contains a full separate copy of the project with its own src/, dist/, package.json, and extension/ folder. Meanwhile prompt-lab-source/extension/ is another copy. Three copies of background.js, options.*, and manifest.json exist at different paths with different contents.
Fix → Flatten to a single source tree. Remove the nested prompt-lab-extension/ directory. Keep prompt-lab-source/extension/ as the canonical extension file location. The assembler already copies from there.
Medium Misleading message type name
ANTHROPIC_REQUEST is the message type for all 5 providers. MODEL_REQUEST is accepted as an alias but never sent by any code path. Anyone reading the source will assume this only works with Anthropic.
Fix → Update api.js to send MODEL_REQUEST. Keep ANTHROPIC_REQUEST in background.js listener array for backward compat with old compiled panel JS.
Medium Monolithic App.jsx (2000 lines, 40+ state vars)
Every feature — editor, composer, A/B test, library, pad, wizard, settings, command palette — lives in a single component. Adding per-feature provider selection, streaming, or any new tab will make this worse. State entanglement means a bug in the pad can theoretically affect the editor.
Fix → Extract: useProvider hook, SettingsModal.jsx, OnboardingWizard.jsx, useABTest hook. Keep App.jsx as layout shell + tab router. Do this before the next major feature.
Medium errorTaxonomy.js only handles Anthropic error shapes
Gemini returns SAFETY blocks, OpenRouter returns moderation errors, OpenAI returns insufficient_quota. All of these fall through to a generic message. Users won't understand why their request was blocked.
Fix → Add provider-specific error patterns to errorTaxonomy.js. At minimum: Gemini safety block, OpenAI quota/rate limit, OpenRouter moderation.
Medium Inconsistent localStorage key prefix
Most keys use pl2- prefix. The pad uses pl-pad (no "2"). Minor but creates ambiguity about which version of the app owns which keys.
Fix → Rename to pl2-pad with a migration shim that reads the old key on first load.
Low Web mode only supports Anthropic
api.js web fallback hardcodes Anthropic. Anyone running the panel in a browser during development can only use Anthropic.
Fix → Add getWebProviderConfig() that reads provider/key/model from localStorage. Low priority since most usage is in extension mode.
Low OpenRouter model slug has no validation
Free-text input. A typo like anthropic/clade-sonnet will produce a runtime error with no helpful message.
Fix → Add a lightweight check: if the response is a 404 or model-not-found error, surface "Model slug may be incorrect" instead of the raw error.
Low Static model dropdowns will go stale
Anthropic, OpenAI, and Gemini model lists in options.html are hardcoded. When new models ship, the dropdown won't include them.
Fix → Either fetch model lists dynamically from each provider's API, or add a "Custom model" text input fallback to each dropdown section.
