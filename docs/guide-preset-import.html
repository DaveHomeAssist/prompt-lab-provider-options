Importing Preset Packs

Bring in a curated library of prompts, chains, and input templates — in seconds, no account required.

⏱ 5 min read 🗂 Works in Chrome Extension & Desktop

Overview

Preset packs are portable prompt libraries you can share, import, and fork — think of them as a curated playlist for your AI workflow.

What is a preset pack?

A preset pack is a single .json file that bundles one or more of the following:

💬 Prompts Individual prompts with titles and optional tags
⛓️ Chains Multi-step prompt sequences that run in order
🎛️ Input templates Variable slots your team fills in at run time
📋 Metadata Pack name, version, author, and description

Why use preset packs?

  • Get started instantly — drop in a community pack or a colleague's export and have a full working library in seconds.
  • Stay in sync — when someone updates the pack and shares a new file, re-import to pick up changes.
  • No account required — packs are plain JSON files. Email them, put them in Notion, drop them in Slack.
  • Safe by default — imports never overwrite existing prompts; duplicates are flagged so you decide what to keep.

Tools Needed

Almost nothing. Here's what you'll want before you start.

Requirements

  • Prompt Lab Chrome Extension (v0.4.0 or later) or the Desktop app (any version)
  • A preset pack .json file — sent by a colleague, downloaded from a community source, or exported from another Prompt Lab install
  • That's it

Accepted formats

  • .json file dropped directly onto the import panel
  • Raw JSON text pasted into the paste input
  • A Notion page or database export (see tip below)
Notion users: If your prompts live in a Notion database, use the Prompt Lab Notion Bridge to export them as a preset pack. The bridge handles formatting, tags, and duplicate detection automatically before handing you a ready-to-import JSON file.

Step by Step Instructions

Two ways in — file drop or paste. Both end up in the same place.

Method 1 — Drop a file

Open the Library panel

Click the Prompt Lab extension icon in your browser toolbar (or open the Desktop app). Navigate to the Library tab — it's the bookshelf icon in the left sidebar.

Click "Import Pack"

Look for the Import Pack button near the top-right of the Library panel. It has a small upload arrow icon. Clicking it opens the Import dialog.

Drop your file onto the drop zone

Drag your .json file from Finder (or Explorer on Windows) and drop it onto the dashed drop zone in the dialog. The zone will highlight blue when your file is hovering over it.

You can also click Browse files to open a standard file picker instead.

Review the pack summary

Prompt Lab reads the file and shows you a preview: pack name, author, how many prompts and chains are included, and a list of any duplicate candidates — items that closely match prompts you already have.

Handle duplicates (if any)

For each duplicate candidate, you'll see a side-by-side diff. Choose Skip to keep your existing version, Replace to overwrite with the pack version, or Keep Both to import the new one alongside the existing one with a suffix.

Heads up: "Replace" is permanent — it overwrites your existing prompt text. If you're not sure, choose "Keep Both" and clean up later.

Click "Import"

Hit the Import button at the bottom of the dialog. Prompt Lab adds everything to your library and shows a confirmation toast with a count of what was imported.

Find your new prompts

Imported prompts appear in your Library immediately. They'll have the pack name as a tag so you can filter by it. Use the search bar or click the pack tag to see only the newly imported items.

Method 2 — Paste JSON

Open the Import dialog and switch to Paste mode

Follow steps 1–2 above, then click the Paste JSON tab at the top of the Import dialog.

Paste your JSON

Copy the full contents of a preset pack JSON file and paste it into the text area. Prompt Lab validates the format as you paste — a green checkmark means it's valid, a red outline means something's off.

Quick copy trick: If a colleague sent you a preset pack in Notion or Google Docs, select all the text in the code block (Cmd+A), copy it, and paste it here. Prompt Lab strips any surrounding whitespace automatically.

Continue from step 4

From this point the flow is identical to the file drop method — review the summary, handle duplicates, and click Import.

Using select-type input variables

What are select inputs?

Some preset packs include prompts with dropdown-style variables — for example, a writing prompt that lets you pick a tone from a list like formal, casual, punchy. These are called select inputs.

They import like everything else

Select inputs come in automatically with the pack. No extra steps. When you run a prompt that has a select input, Prompt Lab renders a dropdown instead of a free-text field.

Editing options after import

To change the available options in a select input, open the prompt in the editor, find the variable in the Inputs panel, and click the pencil icon next to it. You can add, remove, or reorder options from there.

Tips

Little things that save time and confusion.

Tag filtering is your friend. Every imported prompt gets automatically tagged with the pack name. After importing, filter by that tag to see exactly what came in — then reassign items to your own tags as needed.
You can import the same pack more than once. Prompt Lab checks for duplicates on each import, so re-importing an updated pack is safe. New items come in; existing ones are flagged for your review.
Packs work across platforms. A pack exported from the Chrome Extension imports cleanly into the Desktop app and vice versa. The format is identical.
Chains preserve step order. If the pack includes a multi-step chain, the steps arrive in the correct sequence. You don't need to reorder anything after import.
Want to create your own pack? Go to Library → Export, select the prompts you want to bundle, and click Export as Pack. The output file follows the same schema and is ready to share immediately.
Version field helps you track updates. Preset packs include a version field. Ask the person sharing the pack to bump the version when they update it — that way you'll always know which version you have and whether a re-import might bring something new.

Common Mistakes

If something's not working, one of these is probably why.

📄

Pasting only part of the JSON

A preset pack JSON file starts with { and ends with }. If you copied from a document that truncated the content or cut off mid-line, the validator will reject it. Always copy the entire file — open it in a text editor and use Cmd+A to make sure you got everything.

🔑

Missing required fields

A valid preset pack must include version, type, and at least one of presets, chains, or inputs. Packs hand-crafted in a text editor sometimes skip these. Prompt Lab will tell you exactly which field is missing in the validation error message.

🔄

Accidentally replacing a prompt you wanted to keep

When a duplicate is detected, "Replace" is a one-way door — the old prompt text is overwritten. If you're unsure whether the incoming version is better, always choose "Keep Both" first. You can delete the extra copy later once you've compared them.

📦

Trying to import a raw Notion export

Notion's native CSV and Markdown exports are not preset packs. You need to run the Notion Bridge first — it converts your Notion database into the correct schema. See the Notion Bridge setup guide for details.

If the Import button stays greyed out after dropping a file: the file failed validation. Click the See errors link in the dialog to read the specific issue. The most common cause is a stray trailing comma in the JSON — a text editor with JSON linting (VS Code, for example) will highlight it instantly.
Don't import packs from sources you don't trust. Preset packs can contain prompt text designed to jailbreak models or exfiltrate information through your outputs. Stick to packs from colleagues, verified community sources, or packs you created yourself.
