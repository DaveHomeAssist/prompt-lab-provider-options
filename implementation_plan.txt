Prompt Lab Implementation Plan
Date: 2026-03-24
Prepared from: AGENTS.md issue tracker, CURRENT_PROJECT_REPORT.md, and UX_AUDIT_2026-03-17.md

Plan Scope
- Primary task: resolve the two active product issues still listed as in progress.
- Issue 002: Create workflow remains too vertically stacked.
- Issue 003: Experiments and run history are still split.
- Constraint: preserve the current multi-provider abstraction, local-first persistence, and shared frontend architecture.

Delivery Model
- Total timeline: 6 weeks
- Release style: phased rollout behind stable UI states, with targeted Vitest coverage after each phase
- Assumption: one product owner can hold multiple roles if the team is small

Roles and Responsible Parties
- Product Owner / UX Lead
  - Owns workflow decisions, acceptance criteria, and release signoff
- Frontend Engineer
  - Owns React/Vite implementation in prompt-lab-extension shared frontend
- QA / Accessibility Reviewer
  - Owns regression coverage, keyboard flow checks, and responsive verification
- Release Owner
  - Owns web, desktop, and extension smoke validation before release
- AI Support (Codex)
  - Assists with implementation, refactors, and test creation under human review

Key Success Criteria
- Create flow requires fewer scroll-heavy steps to enhance, save, and iterate
- Experiments and run history are reachable from one coherent Evaluate surface
- No regressions across extension, desktop, and web shells
- Keyboard and focus behavior remain WCAG-oriented and predictable
- Existing A/B, library, and persistence flows continue to work

Cross-Phase Dependencies

- Phase 2 depends on Phase 1 layout and save-flow decisions
- Phase 3 depends on the agreed navigation model from Phase 0 and stable Create changes from Phases 1-2
- Phase 4 depends on completion of the major UI changes in Phases 1-3
- Phase 5 depends on successful verification in Phase 4

Risks and Mitigations
- Risk: shared frontend changes break multiple shells
  - Mitigation: keep all major changes inside the shared shell entry points and smoke-test web plus extension
- Risk: IA cleanup hides existing expert workflows
  - Mitigation: preserve sub-views and command access while reducing top-level choices
- Risk: Evaluate refactor breaks persisted run data expectations
  - Mitigation: keep storage schemas stable and limit changes to navigation/state composition first
- Risk: vertical stacking returns in compact mode
  - Mitigation: validate narrow widths during each phase, not only at the end

Recommended Checkpoints
- End of Week 1: demo Create compression before continuing
- End of Week 2: approve Create polish and test coverage
- End of Week 4: demo unified Evaluate surface with persisted state
- End of Week 5: accessibility and regression signoff
- End of Week 6: release go/no-go review
